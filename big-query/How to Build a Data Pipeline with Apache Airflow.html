How to Build a Data Pipeline with Apache Airflow

Apache Airflow is an open-source workflow management tool designed for ETL/ELT (extract, transform, load/extract, load, transform) workflows. It enables users to define, schedule, and monitor complex workflows, with the ability to execute tasks in parallel and handle dependencies between tasks.

I have used it for different workflows, from the simplest to complex and scalable, and in both cases it has many advantages and also details to take care of.

Apache Airflow has a modular architecture, making it easy to add new operators and integrations with external systems. Additionally, it provides a web-based UI for visualizing and monitoring workflow execution. It also includes a rich set of APIs and command-line tools for managing workflows programmatically.

One key benefit of Apache Airflow is its ability to handle complex workflows with ease. This includes workflows involving multiple data sources, transformations, and destinations. Additionally, it provides a robust scheduling engine that can handle failures and retries, ensuring that workflows are completed reliably and on time.

Apache Airflow 2.0 is the latest version and I invite you to review its official documentation. This new version brings many improvements and new features that make the tool even more powerful and user-friendly for managing ETL/ELT workflows.

One of the major changes in Apache Airflow 2.0 is the redesigned UI, which is more intuitive and user-friendly. The new UI includes a drag-and-drop interface for creating workflows and a graph view that makes it easier to visualize and manage complex workflows.

Apache Airflow 2.0 introduces several major improvements, including support for cloud-native features like Kubernetes. This allows for easy deployment and scaling of Airflow on cloud platforms such as Google Cloud Platform and Amazon Web Services. For me, this is the best functionality because it allows you to decide which cloud you should work in and takes work away from having to solve it on your part as was done in the previous version.

In addition, Apache Airflow 2.0 includes new integrations with popular data warehousing and processing tools like Snowflake and Databricks. This makes it easier to manage complex data pipelines across multiple platforms.

What are the steps to write a data pipeline?

Building a data pipeline with Apache Airflow involves the following steps:

  1. Define the workflow: Determine the tasks that need to be performed, their dependencies, and the order in which they should be executed. This can be done using Python scripts, which define the tasks and their relationships.
  2. Configure the environment: Install and configure Apache Airflow, including setting up the database backend and other dependencies. This can be done using a Docker container or by installing it on a virtual machine or server.
  3. Create the DAG: Define the Directed Acyclic Graph (DAG) that represents the workflow in Apache Airflow. This can be done using the web-based UI or by writing the DAG definition in Python code.
  4. Define the tasks: Create the individual tasks that make up the workflow, using the built-in operators provided by Apache Airflow or custom operators if needed. These tasks can perform a variety of functions, such as extracting data from a source, transforming it, and loading it into a destination.
  5. Set up the connections: Configure connections to external systems such as databases, APIs, or cloud services, so that the tasks can interact with them as needed.
  6. Test the workflow: Test the workflow by running it in a development or staging environment, and verifying that the tasks are executed correctly and in the right order.
  7. Deploy the workflow: Deploy the workflow to a production environment, where it can be run on a schedule or triggered manually as needed.

The following is a small example of a DAG I made:

Now the code which is very simple but the idea is to exemplify.

from datetime import datetime, timedelta
import pandas as pd
from time import sleep

from airflow import DAG
from airflow.operators.python_operator import PythonOperator


def send_msg_python():
    print("Just a testing message")

def create_csv_python(temp_file):
    df = pd.DataFrame({'id': [1, 2, 3], 'name': ['John', 'Peter', 'Marie'], 'age': [30, 25, 35]})
    df.to_csv(temp_file, index=False)

def waiting_python():
    sleep(15)

def parser_csv_python():
    df = pd.read_csv('/tmp/data_etl.csv')
    df = df[['id', 'name']]
    print(df)
    df.to_csv('/tmp/data_result.txt', sep='|', index=False, header=False)

default_args = {
    'owner': 'Yesi Days',
    'depends_on_past': False,
    'start_date': datetime(2023, 2, 22),
    'email': ['me.d@something.com'],
    'email_on_failure': False,
    'email_on_retry': False,
    'retries': 1,
    'retry_delay': timedelta(minutes=5),
}

dag = DAG(
    'etl_dag',
    default_args=default_args,
    description='ETL Example',
    schedule_interval=timedelta(days=1),
)

send = PythonOperator(
    task_id='send_msg',
    python_callable=send_msg_python,
    dag=dag,
)

wait = PythonOperator(
    task_id='waiting',
    python_callable=waiting_python,
    dag=dag,
)

create = PythonOperator(
    task_id='create_csv',
    python_callable=create_csv_python,
    op_kwargs={'temp_file': '/tmp/data_etl.csv'},
    dag=dag,
)

parser = PythonOperator(
    task_id='parser_csv',
    python_callable=parser_csv_python,
    dag=dag,
)

send >> create >> wait >> parser

Finally, the grid list where you can see how many times it was executed, when it failed, tasks waiting, among others. This is one of the best views because it helps you understand the flow in time and tasks.

As you can see, Apache Airflow provides a powerful and flexible platform for building and managing data pipelines. Whether you’re working with large volumes of structured or unstructured data, or need to integrate multiple data sources into a single workflow, Apache Airflow provides the flexibility and functionality necessary to get the job done.
