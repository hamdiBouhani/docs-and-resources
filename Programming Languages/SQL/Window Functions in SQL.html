Window Functions in SQL

Have you ever heard about window functions in SQL? Window functions have being popular in data analysis and business intelligence sector since a long time ago. Window functions are very impactful in data analysis (also business intelligence) and are mostly used in SQL queries to perform calculations on specific groups of data within a larger dataset. We can calculate aggregate functions (MAX, MIN, SUM, etc.) over a specific set of rows, rather than the entire dataset.

What exactly are window functions?

Window functions or analytic functions are a SQL function that executes on a subset of rows in a dataset, rather than on the entire dataset. Window functions can be used to perform calculations over a specific range of rows or to calculate values based on the values in other rows in the dataset.

How do window functions work?

Window functions are applied to a group of rows referred to as a window or frame. The window is defined using a partition clause (we also can defined it without a partition clause, based on our needs) which divides the data into subsets which specifies the order of the rows within each subset. The window function then applies an operation to the rows within the window. Window functions have a lot of types, for instances:

• Aggregate functions

Perform calculation on a value based on a subset of rows within the window, including MAX, MIN, SUM, AVG, and COUNT.

• LEAD and LAG functions

Return a value from a row that is either one row ahead or one row behind the current row within the window. We can also set how many rows will be skipped and return a desired output ( varchar, int, decimal, etc.) by inputting a value in its parameter.

• Ranking functions

Assign a rank or row number to each row within the window based on a specified order, including ROW_NUMBER, RANK, and DENSE_RANK.

An Example of Window Functions

For this example, I have a table named customer_w_country containing customer_id, full_name, and country. I want to know the first three customers (based on customer_id) from each country to give them a reward because of their loyalty. So I am going to perform the window function using rank() function to provide the solution.

-- Query :
select * from (
 select a.*, 
 rank() over(partition by country order by customer_id) as rk
 from customer_w_country a) as x
where x.rk < 4;

And we can see the output below:

As we can see, we can see the first three customers from each country, such as Canada, Chile, and China.

Conclusion

Window functions are an effective way to do calculations on subsets of data within a larger dataset.We can use window functions to compute aggregate functions, assign rankings, compute running totals or averages, and so on. We can increase the efficiency and effectiveness of our data analysis queries by learning how window functions work and their different use cases.
